## 🔧 高级调试技巧

### 使用浏览器开发者工具

1. **打开开发者工具**：按 `F12`
2. **监控网络请求**：切换到 `Network` 面板
3. **查看缓存日志**：切换到 `Console` 面板
4. **模拟离线**：`Network` → 勾选 `Offline`

### Console 日志说明

在新闻详情页，Console 会显示：
- `发起网络请求，id: X` - 首次加载，使用网络
- `使用缓存数据，id: X` - 再次加载，使用缓存

## 📝 完整验证流程

### 步骤 1：基础测试
1. 打开检查页面
2. 依次点击 4 个测试按钮
3. 确认所有测试通过 ✅

### 步骤 2：新闻列表验证
1. 点击「📖 查看新闻列表」
2. 观察列表加载
3. 下拉刷新测试
4. 点击任意新闻进入详情

### 步骤 3：缓存策略验证
1. 在新闻详情页查看 Console
2. 应该看到 `发起网络请求`
3. 返回列表，再次点击同一条新闻
4. Console 应该显示 `使用缓存数据`

### 步骤 4：离线阅读验证
1. 先在线查看几条新闻（确保已缓存）
2. 打开 Chrome DevTools → Network → 勾选 `Offline`
3. 尝试刷新列表 → 应该提示无网络
4. 点击已看过的新闻 → 应该能正常显示（离线阅读）
5. 点击未看过的新闻 → 应该提示加载失败

### 步骤 5：错误处理验证
1. 断网状态下操作
2. 观察错误提示是否友好
3. 确认重试按钮可用

## ⚠️ 常见问题

**Q: 检查页面白屏？**
A: 检查 pages.json 配置是否正确，确保 check 页面在第一项。

**Q: 网络请求失败？**
A: 确认网络连接正常，JSONPlaceholder 服务可访问。

**Q: 缓存不生效？**
A: 检查浏览器是否禁用了本地存储 (LocalStorage)。

**Q: 如何修改检查页面？**
A: 编辑 pages/check/check.vue 文件。
